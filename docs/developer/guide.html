    Developer Guide

    This guide covers GoBlocks's architecture, PHP extension points, CSS token system, REST API client usage, and the build process for contributing to the plugin.

    Adding Custom Dynamic Tags

    Register new tags via the goblocks_dynamic_tags filter. Each tag is a PHP callable keyed by its slug.

    <?php
add_filter( 'goblocks_dynamic_tags', function( array $tags ): array {
    // {reading_time} — estimated reading time in minutes
    $tags['reading_time'] = function( array $options, WP_Post $post ): string {
        $words   = str_word_count( strip_tags( $post->post_content ) );
        $minutes = (int) ceil( $words / 200 );
        $suffix  = esc_html( $options['suffix'] ?? 'min read' );
        return $minutes . ' ' . $suffix;
    };

    // {acf_field} — read an Advanced Custom Fields value
    $tags['acf_field'] = function( array $options, WP_Post $post ): string {
        if ( ! function_exists( 'get_field' ) ) {
            return esc_html( $options['fallback'] ?? '' );
        }
        $key   = sanitize_key( $options['key'] ?? '' );
        $value = get_field( $key, $post->ID );
        return esc_html( $value ?: ( $options['fallback'] ?? '' ) );
    };

    return $tags;
} );

    Usage in the editor:

    {reading_time}
{reading_time|suffix:minutes}
{acf_field|key:product_sku|fallback:N/A}

    Modifying Query Block Arguments

    Use the goblocks_query_args filter to modify the WP_Query arguments for any Query block.

    <?php
add_filter( 'goblocks_query_args', function( array $args, array $block_attrs ): array {
    // Only modify queries for a specific post type
    if ( ( $args['post_type'] ?? '' ) === 'product' ) {
        // Order WooCommerce products by menu_order (custom sort)
        $args['orderby']   = 'menu_order';
        $args['order']     = 'ASC';
        // Only show featured products
        $args['tax_query'] = [
            [
                'taxonomy' => 'product_visibility',
                'field'    => 'name',
                'terms'    => 'featured',
            ],
        ];
    }
    return $args;
}, 10, 2 );

    Modifying Block CSS Output

    Use the goblocks_block_css filter to append or modify the CSS generated for any block.

    <?php
add_filter( 'goblocks_block_css', function( string $css, array $block, WP_Post $post ): string {
    if ( $block['blockName'] !== 'goblocks/group' ) {
        return $css;
    }
    // Append a print-specific style to every Group block
    $uid = esc_attr( $block['attrs']['uid'] ?? '' );
    $css .= "@media print { .gb-block-{$uid} { page-break-inside: avoid; } }";
    return $css;
}, 10, 3 );

    Registering Custom Patterns

    <?php
add_action( 'init', function(): void {
    register_block_pattern(
        'my-plugin/hero-dark',
        [
            'title'      => __( 'Dark Hero Section', 'my-plugin' ),
            'categories' => [ 'goblocks' ], // appears in GoBlocks category
            'content'    => '<!-- wp:goblocks/group {"backgroundColor":"#17182B"} -->…<!-- /wp:goblocks/group -->',
        ]
    );
} );

    CSS Token Reference

    GoBlocks uses a three-tier CSS custom property system. All properties start with --gb-.

    Tier 1 — Primitive tokens (raw values):

    --gb-color-blue-500: #3858E9;
--gb-color-cyan-400: #5CE0E6;
--gb-space-4: 1rem;
--gb-space-8: 2rem;
--gb-radius-md: 8px;

    Tier 2 — Semantic tokens (role-based, reference primitive tokens):

    --gb-color-brand: var(--gb-color-blue-500);
--gb-color-accent: var(--gb-color-cyan-400);
--gb-color-surface: #ffffff;
--gb-color-text: #17182B;
--gb-container-site: 1200px;

    Tier 3 — Component tokens (block-specific, reference semantic tokens):

    --gb-btn-bg: var(--gb-color-brand);
--gb-btn-radius: var(--gb-radius-md);
--gb-card-shadow: 0 2px 16px rgba(0,0,0,.08);
--gb-tab-active-color: var(--gb-color-brand);

    Accessing Settings in PHP

    <?php
$settings        = get_option( 'goblocks_settings', [] );
$container_width = $settings['container_width'] ?? 1200;
$css_method      = $settings['css_print_method'] ?? 'file';
$palette         = $settings['global_color_palette'] ?? [];

// Iterate the color palette
foreach ( $palette as $color ) {
    // $color['slug'], $color['name'], $color['color']
    printf( '--gb-color-%s: %s;', esc_attr( $color['slug'] ), esc_attr( $color['color'] ) );
}

    To use GoBlocks's settings class directly:

    <?php
use GoBlocks\Core\Plugin;
$settings = Plugin::get_settings(); // returns typed Settings object
$width    = $settings->container_width;

    REST API Access from JavaScript

    const response = await fetch( '/wp-json/goblocks/v1/settings', {
  method: 'GET',
  headers: {
    'Content-Type': 'application/json',
    'X-WP-Nonce': wpApiSettings.nonce,
  },
} );
const settings = await response.json();
console.log( settings.container_width ); // 1200

    All endpoints require the user to be authenticated with manage_options capability. See REST API Reference for the full endpoint list.

    Build from Source

    Prerequisites

    • Node.js 20+ and npm 10+
    • PHP 8.0+ and Composer
    • WP-CLI (for development environment management)

    Setup

    git clone https://github.com/godevsltd/goblocks.git
cd goblocks

# Install PHP dependencies
composer install

# Install JavaScript dependencies
npm install

    Build Commands

    # Development build with file watcher
npm run start

# Production build (minified, no source maps)
npm run build

# Type check TypeScript
npm run type-check

# Lint PHP (WPCS + PHPStan)
composer run lint

# Run PHP tests
composer run test

    Output

    Built JavaScript and CSS files are output to assets/js/ and assets/css/. These compiled files are committed to the repository and are what gets distributed in the plugin ZIP.
