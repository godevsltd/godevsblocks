    REST API Reference

    GoBlocks registers 14 REST API endpoints under the namespace goblocks/v1. All endpoints require the user to be authenticated and have the manage_options capability.

    Base URL: /wp-json/goblocks/v1/

    Authentication: Include the WordPress REST nonce in the X-WP-Nonce request header. In the block editor, the nonce is available at wpApiSettings.nonce.

    Settings Endpoints

    GET /settings

    Returns the full GoBlocks settings object.

    Response:

    {
  "container_width": 1200,
  "css_print_method": "file",
  "sync_responsive": true,
  "disable_google_fonts": false,
  "enable_dark_mode": false,
  "breakpoints": { "xs": 480, "sm": 640, "md": 768, "lg": 1024, "xl": 1280, "2xl": 1536 },
  "global_color_palette": [],
  "global_typography": []
}

    POST /settings

    Updates one or more settings. Only keys included in the request body are updated; omitted keys retain their current values.

    Request body:

    {
  "container_width": 1440,
  "css_print_method": "inline"
}

    Response: Updated full settings object.

    POST /settings/reset

    Resets all settings to their default values. No request body required.

    Response: Default settings object.

    Global Styles Endpoints

    GET /styles

    Returns the current global color palette and typography presets as a CSS custom property string ready to inject.

    Response:

    {
  "css": ":root { --gb-color-primary: #3858E9; --gb-color-accent: #5CE0E6; }",
  "palette": [ { "slug": "primary", "name": "Brand Blue", "color": "#3858E9" } ],
  "typography": []
}

    POST /styles

    Updates the global color palette and/or typography presets.

    Request body:

    {
  "palette": [
    { "slug": "primary", "name": "Brand Blue", "color": "#3858E9" },
    { "slug": "accent", "name": "Cyan", "color": "#5CE0E6" }
  ]
}

    Response: Updated styles object.

    CSS Cache Endpoints

    GET /css/cache

    Returns information about the current CSS cache: total files, total size, and a list of cached post IDs.

    Response:

    {
  "total_files": 42,
  "total_size_bytes": 284672,
  "cached_post_ids": [1, 5, 12, 42, 107]
}

    DELETE /css/cache

    Deletes all CSS cache files. CSS will be regenerated on the next frontend request for each page.

    Response:

    { "deleted": 42 }

    DELETE /css/cache/{post_id}

    Deletes the CSS cache file for a specific post ID.

    Path parameter: {post_id} — integer post ID

    Response:

    { "deleted": true, "post_id": 42 }

    Pattern Endpoints

    GET /patterns

    Returns a list of all registered GoBlocks patterns with their metadata (no block content).

    Response:

    [
  {
    "name": "goblocks/hero-centered",
    "title": "Centered Hero",
    "categories": ["goblocks"],
    "keywords": ["hero", "header"]
  }
]

    GET /patterns/{name}

    Returns a single pattern by its registered name, including its full block content string.

    Path parameter: {name} — URL-encoded pattern name, e.g. goblocks%2Fhero-centered

    POST /patterns

    Registers a new pattern programmatically. Equivalent to register_block_pattern().

    Request body:

    {
  "name": "my-plugin/dark-hero",
  "title": "Dark Hero",
  "categories": ["goblocks"],
  "content": "<!-- wp:goblocks/group -->…<!-- /wp:goblocks/group -->"
}

    DELETE /patterns/{name}

    Unregisters a pattern. Equivalent to unregister_block_pattern(). Only patterns registered via the REST API can be deleted this way — built-in GoBlocks patterns cannot be removed.

    Block Metadata Endpoints

    GET /blocks

    Returns metadata for all 36 registered GoBlocks block types: name, title, description, category, and attribute schema.

    Response excerpt:

    [
  {
    "name": "goblocks/group",
    "title": "Group",
    "description": "A flexible layout container.",
    "category": "goblocks",
    "attributes": { "tag": { "type": "string", "default": "div" } }
  }
]

    GET /blocks/{name}

    Returns full metadata for a single block type.

    Path parameter: {name} — URL-encoded block name, e.g. goblocks%2Fgroup

    Error Responses

    All endpoints return standard WordPress REST API error responses:

    {
  "code": "rest_forbidden",
  "message": "Sorry, you are not allowed to do that.",
  "data": { "status": 403 }
}
    HTTP StatusMeaning
    200Success
    400Bad request (invalid parameters)
    401Authentication required (not logged in)
    403Forbidden (logged in but lacks manage_options)
    404Resource not found (e.g. pattern name does not exist)
    500Server error (e.g. unable to write CSS file)
