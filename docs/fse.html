    Full Site Editing

    GoBlocks is fully compatible with the WordPress Site Editor (FSE). All 36 blocks and 41 patterns are available in the Site Editor, and GoBlocks integrates with theme.json, template CSS generation, and the Global Styles panel.

    Requirements

    Full Site Editing requires an FSE-compatible theme — a theme that includes a theme.json file and a /templates folder. Compatible themes include:

    • Twenty Twenty-Four and Twenty Twenty-Five (bundled with WordPress)
    • Any block theme in the WordPress.org theme directory
    • Custom block themes with "blockTheme": true in theme.json

    Classic themes (PHP template files, no theme.json) do not support the Site Editor. GoBlocks blocks still work in the post editor on classic themes, but template-level editing is not available.

    Opening the Site Editor

    1. Go to Appearance → Editor (visible only when an FSE theme is active).
    2. The Site Editor opens. The left panel shows Templates, Template Parts, and Patterns.
    3. Click any template (e.g. Single Post) to edit it.
    4. Insert GoBlocks blocks anywhere in the template the same way you would in the post editor.

    Using GoBlocks Blocks in Templates

    All GoBlocks blocks are available in templates. Most are useful for static decoration — layouts, dividers, heading sections. For dynamic post data in templates, use the Dynamic Content tag system.

    Typical single-post template structure:

    Template: Single Post
  └── goblocks/group  (hero section)
       ├── goblocks/heading  {post_title}
       └── goblocks/text     {post_date|format:M j, Y} · {author_name}
  └── core/post-content
  └── goblocks/group  (related posts section)
       └── goblocks/query
            └── goblocks/query-loop
                 ├── goblocks/image    {featured_image|size:medium}
                 └── goblocks/heading  {post_title|linked:true}

    Template-Level CSS

    GoBlocks generates per-page CSS files (see Performance). For templates:

    • Each template type (single, archive, front-page) gets its own CSS file.
    • Template CSS is cached in wp-content/uploads/goblocks/template-{slug}.css.
    • CSS is regenerated when the template is saved in the Site Editor.

    theme.json Integration

    GoBlocks reads your theme's theme.json to align its design tokens:

    • Color palette: Colors defined in theme.json under settings.color.palette are available in GoBlocks color pickers, merged with the GoBlocks Global Styles palette.
    • Font sizes: Font size presets from settings.typography.fontSizes are available in GoBlocks Typography controls.
    • Spacing scale: Spacing presets from settings.spacing.spacingSizes are offered as quick-pick options in padding/margin controls.
    • CSS custom properties: GoBlocks injects its own --gb-* CSS custom properties alongside the --wp--preset--* properties from theme.json. Both namespaces are available in your custom CSS.

    Example: If your theme defines a primary color in theme.json:

    {
  "settings": {
    "color": {
      "palette": [
        { "slug": "primary", "color": "#3858E9", "name": "Primary" }
      ]
    }
  }
}

    WordPress generates --wp--preset--color--primary: #3858E9. GoBlocks color pickers will show "Primary" and let you select it.

    Global Styles Panel

    When using an FSE theme, GoBlocks settings are accessible inside the WordPress Global Styles panel (the paintbrush icon in the Site Editor toolbar). This provides a unified design tokens UI instead of requiring a separate admin panel.

    • Color palette, typography, and spacing presets from GoBlocks appear in the Global Styles panel.
    • Changes saved in the Global Styles panel are synced back to GoBlocks settings.

    Dynamic Content Tags in Templates

    Dynamic content tags work in templates just as they do in the post editor. The tags resolve at render time relative to the post or archive being rendered.

    On single post templates: All post data, author, and featured image tags resolve for the current post.

    On archive templates: The Query block powers the loop; tags inside the Query Loop resolve for each post in the loop. Term-level tags ({term_name}, {term_description}) resolve for the current archive term.

    On static templates (front page, 404, search): Post-specific tags that require a post context will return empty strings. Use fallback options for these contexts:

    {post_title|fallback:Welcome}

    Pattern Library in the Site Editor

    All 41 GoBlocks patterns are available in the Site Editor's Patterns panel.

    1. In the Site Editor, click the + block inserter.
    2. Switch to the Patterns tab.
    3. Select the GoBlocks category.
    4. Click any pattern to insert it into the template.

    You can also create custom synced patterns (formerly reusable blocks) from any GoBlocks block or group — right-click the block and select Create pattern. Synced patterns update everywhere they are used when you edit the pattern.

    Template Parts

    GoBlocks blocks work inside WordPress template parts (header, footer, sidebar). Template parts are reusable fragments that can be embedded across multiple templates.

    Recommended uses:

    • Header part: GoBlocks Navigation block for the site-wide header.
    • Footer part: GoBlocks Group + Text for footer layout and copyright notice using {current_year} and {site_name} tags.
    • CTA part: A GoBlocks Group with a call-to-action section inserted in multiple page templates.
