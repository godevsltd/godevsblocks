    Frequently Asked Questions

    General

    Is GoBlocks free?

    Yes. GoBlocks is free and open source, licensed under GPL-2.0-or-later. It is available on WordPress.org and all features documented here are included at no cost.

    Does GoBlocks require a specific theme?

    No. GoBlocks works with any WordPress theme that runs WordPress 6.5+. For the Site Editor and template editing, a Full Site Editing (block theme) is required.

    Does GoBlocks work with the Classic Editor plugin?

    No. GoBlocks blocks are registered for the block editor (Gutenberg). The Classic Editor plugin disables the block editor, making GoBlocks blocks unavailable for editing. The blocks will still render on the frontend if content was saved before switching to the Classic Editor.

    Can I use GoBlocks alongside Elementor or Divi?

    GoBlocks is a Gutenberg block plugin and does not conflict with Elementor or Divi at the PHP level. However, Elementor and Divi replace the WordPress block editor with their own builders. GoBlocks blocks are not available in those builders — they are only available in the WordPress block editor.

    Does GoBlocks support WordPress Multisite?

    Yes. GoBlocks is fully multisite-compatible. Each site in the network has its own settings and CSS cache. See Installation → Multisite for details.

    Blocks

    How do I find GoBlocks blocks in the editor?

    Click the + block inserter. Scroll to the GoBlocks category, or type gb in the search field to filter all GoBlocks blocks.

    Can I nest GoBlocks blocks inside core WordPress blocks?

    Yes, in most cases. GoBlocks Group, Column, and Container blocks accept any child block including core blocks. Core blocks like the Group or Columns block also accept GoBlocks blocks as children.

    Can I create my own block?

    GoBlocks provides a framework for extending blocks via PHP filters. To add an entirely custom block type, use the standard WordPress block registration API (register_block_type()) — this is independent of GoBlocks.

    Can I deep-link to a specific tab?

    Yes. Add a URL hash that matches the tab's ID (set in the Tab Panel's Advanced settings). When the page loads with a matching hash, the corresponding tab is activated automatically.

    How do I keep an accordion item open by default?

    Select the Accordion Item block, open the Inspector, and enable Open by default. This sets the open attribute on the <details> element.

    Patterns

    How do I edit a GoBlocks pattern after inserting it?

    Click any element inside the inserted pattern to select it, then use the Inspector to modify it. Pattern content is immediately editable after insertion — the original GoBlocks pattern is not modified.

    Can I create my own patterns?

    Yes. Build a layout in the editor → right-click a block → Create pattern. Or register patterns in PHP using register_block_pattern(). See Developer Guide → Custom Patterns.

    My pattern isn't showing in the pattern library. Why?

    Ensure your pattern is assigned to the goblocks or another registered category. Unassigned patterns may appear under Uncategorized in the inserter.

    Styling

    What is the difference between GoBlocks Global Styles and the theme's Global Styles?

    WordPress's Global Styles (the paintbrush icon in the editor) applies to all blocks site-wide via theme.json. GoBlocks's Global Styles (in GoBlocks → Global Styles) applies specifically to GoBlocks blocks and registers colors and typography as CSS custom properties with the --gb-* prefix. On FSE themes, both systems are accessible from the same Global Styles panel.

    Can I add custom CSS to a block?

    Yes. Every GoBlocks block has an Advanced → Additional CSS class field where you can add one or more custom class names. You can then style those classes in your theme's stylesheet or the WordPress Customizer's Additional CSS field.

    My CSS changes are not appearing on the frontend. Why?

    The most common cause is a stale CSS cache. Go to GoBlocks → Settings → Clear CSS Cache and reload the page. If you are using a caching plugin, purge its cache as well.

    Dynamic Content

    Why do I see {post_title} in the editor instead of the actual title?

    Dynamic tags are resolved server-side at render time. In the block editor (which previews content in the browser), the raw tag is shown as a placeholder. The resolved value appears on the actual frontend page.

    Can I use dynamic tags outside a Query Loop?

    Yes. Tags resolve relative to the current page's post when used outside a Query Loop. On a single post page, {post_title} shows that post's title. On an archive or the home page, post-specific tags return empty strings.

    How do I display a custom field value?

    Use the {post_meta} tag with the key option: {post_meta|key:your_field_name}.

    Performance

    Will GoBlocks hurt my PageSpeed score?

    No — GoBlocks is designed to improve PageSpeed scores compared to monolithic page builders. Per-page CSS files are small and browser-cacheable. JavaScript is loaded only when the relevant block is present on the page. The heaviest optional JS (Lottie player) only loads on pages with a Lottie block.

    My CSS file is regenerating on every page load. What is wrong?

    This usually means the uploads directory is not writable, or a server-level cache is preventing GoBlocks from detecting the existing file. See Troubleshooting → CSS regenerating on every load.

    Do I need any special WP Rocket configuration?

    Exclude wp-content/uploads/goblocks/ from WP Rocket's CSS minification and file optimization. CSS files in that directory are already minified by GoBlocks. For all other WP Rocket settings, GoBlocks works with the defaults.

    Compatibility

    Does GoBlocks work with WooCommerce?

    Yes. GoBlocks blocks can be used on WooCommerce pages. The Query block can query product post type with all WooCommerce taxonomy filters. Dynamic tags like {post_title} and {featured_image} resolve for product posts.

    Does GoBlocks work with Cloudflare?

    Yes. Cloudflare serves GoBlocks CSS files from its cache. Cache busting is handled by GoBlocks's version query string, so Cloudflare serves the updated file after a CSS regeneration within the configured cache TTL. If you experience stale CSS after design changes, purge the Cloudflare cache for the affected page's CSS URL.

    Does GoBlocks work with Beaver Builder?

    GoBlocks and Beaver Builder can coexist on the same site if used on different pages. They cannot be used on the same page at the same time, as Beaver Builder replaces the WordPress block editor for pages it manages.
