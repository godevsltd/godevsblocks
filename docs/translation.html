    Translation

    GoBlocks is fully translatable. All user-facing strings are internationalized and available for translation through WordPress.org's GlotPress translation system.

    Text Domain

    GoBlocks uses the text domain goblocks. All PHP strings use __( 'string', 'goblocks' ), _e(), esc_html__(), or their plural equivalents.

    <?php
$label = __( 'Clear CSS Cache', 'goblocks' );

    POT File

    The POT (Portable Object Template) file is the source of all translatable strings. It is located at:

    goblocks/languages/goblocks.pot

    This file is updated on every release. If you are maintaining a translation, update your PO file from the latest POT after each GoBlocks update.

    To regenerate the POT file from source:

    wp i18n make-pot . languages/goblocks.pot \
  --domain=goblocks \
  --exclude=node_modules,vendor

    Translating on WordPress.org (GlotPress)

    The recommended way to contribute translations is through translate.wordpress.org:

    1. Log in with your WordPress.org account.
    2. Go to the GoBlocks translation project.
    3. Select your language.
    4. Translate strings and submit them for review.
    5. Once approved, translations are automatically available to users who have that language set in WordPress settings.

    Tip: Translations on GlotPress require approval by a General Translation Editor for your locale. If you need faster turnaround, use a local file (see below).

    Loco Translate (Recommended for Site Admins)

    The Loco Translate plugin provides a browser-based PO file editor — no FTP or command-line needed.

    1. Install and activate the Loco Translate plugin.
    2. Go to Loco Translate → Plugins → GoBlocks.
    3. Click New language and select your locale.
    4. Translate strings in the browser-based editor.
    5. Click Save — Loco Translate compiles the MO file automatically.

    Manual Translation with Poedit

    1. Download and open Poedit.
    2. Open the POT file: File → New from POT/PO File → select languages/goblocks.pot.
    3. Choose your language and save as goblocks-{locale}.po (e.g. goblocks-fr_FR.po).
    4. Translate strings. When you save, Poedit compiles goblocks-fr_FR.mo automatically.
    5. Upload both .po and .mo files to wp-content/languages/plugins/.

    Translation File Locations

    FileLocation
    POT (source)wp-content/plugins/goblocks/languages/goblocks.pot
    PO (editable translation)wp-content/languages/plugins/goblocks-{locale}.po
    MO (compiled binary)wp-content/languages/plugins/goblocks-{locale}.mo
    JSON (JavaScript strings)wp-content/languages/plugins/goblocks-{locale}-{hash}.json

    Note: Do not place translation files inside wp-content/plugins/goblocks/languages/ — files placed there are overwritten on plugin updates. Always use wp-content/languages/plugins/ for site-specific translations.

    JavaScript Translations

    GoBlocks's block editor scripts use WordPress's JavaScript i18n API:

    import { __ } from '@wordpress/i18n';

const label = __( 'Add Block', 'goblocks' );

    For JavaScript strings to be translated, a JSON translation file must be generated from the PO file and registered with WordPress:

    wp i18n make-json languages/goblocks-fr_FR.po \
  --no-purge \
  --pretty-print

    This creates JSON files in the languages/ directory. GoBlocks registers these files using wp_set_script_translations():

    <?php
wp_set_script_translations(
    'goblocks-editor',
    'goblocks',
    plugin_dir_path( __FILE__ ) . 'languages'
);

    If you are using Loco Translate, it handles JSON file generation automatically.

    RTL (Right-to-Left) Languages

    GoBlocks supports RTL languages (Arabic, Hebrew, Persian, Urdu, etc.):

    • The plugin uses WordPress's built-in RTL detection (is_rtl()).
    • When RTL is active, GoBlocks loads a supplementary goblocks-rtl.css file that mirrors flex directions, text alignment, and margin/padding logical properties.
    • The block editor UI is also RTL-aware.

    To test RTL, set your WordPress site language to Arabic (ar) or Hebrew (he_IL) in Settings → General → Site Language.
