    Dynamic Content

    GoBlocks's dynamic content system lets you display data from the current post, author, taxonomy, or site without writing PHP. Tags are written inline in any rich-text field and resolved at render time on the server.

    Tag Syntax

    {tag_slug}
{tag_slug|option:value}
{tag_slug|option:value|option2:value2}

    Tags are enclosed in { and }. Options are appended after a pipe | as key:value pairs separated by additional pipes.

    Examples:

    {post_title}
{post_date|format:F j, Y}
{post_excerpt|length:20}
{featured_image|size:medium|linked:true}
{post_terms|taxonomy:category|separator:, }

    Which Blocks Support Tags

    Dynamic tags are resolved in the following blocks and fields:

    BlockFields
    TextContent
    HeadingContent
    ButtonLabel, Href, ARIA label
    ImageSrc (URL), Alt text
    AlertContent
    Pricing CardName, Price, Description

    Primary use case: Use dynamic tags inside the Query Loop block to build post card templates — one layout that populates with each post's data automatically.

    Post Data Tags

    TagOutputOptions
    {post_title} The post title linked:true — wraps in permalink anchor
    {post_date} Published date format:M j, Y — PHP date format string
    {post_modified} Last modified date format:M j, Y
    {post_excerpt} Post excerpt (auto-generated if not set) length:30 — word count; more:… — ellipsis text
    {post_content} Full rendered post content length:100 — character limit
    {post_id} Post ID (integer) —
    {post_url} Permalink URL —
    {post_slug} Post slug —
    {post_type} Post type name (e.g. post, page, product) —
    {post_status} Post status (e.g. publish, draft) —
    {post_meta} Value of a custom field key:your_meta_key — required; fallback:N/A

    Author Tags

    TagOutputOptions
    {author_name} Display name of the post author linked:true — links to author archive
    {author_bio} Author biographical info (from profile) length:30 — word count
    {author_avatar} Author Gravatar <img> element size:96 — pixel size
    {author_url} Author archive URL —

    TagOutputOptions
    {featured_image} Full <img> element size:medium — WP image size; linked:true; fallback:/img.jpg
    {featured_image_url} Image URL (no HTML) size:large; fallback:url

    {featured_image_id} Attachment post ID —

    Taxonomy Tags

    TagOutputOptions
    {post_terms} Comma-separated list of terms taxonomy:category — slug required; separator:, ; linked:true
    {post_terms_list} HTML <ul> / <ol> list of terms taxonomy:tag; type:ul or ol; linked:true
    {term_name} First term name (current archive page) taxonomy:category
    {term_description} Term description (current archive) —
    {term_count} Number of posts in the current term —

    Fallback Option

    All tags support a fallback option that is output when the tag resolves to an empty string:

    {post_meta|key:badge_text|fallback:New}
{featured_image_url|size:large|fallback:/images/placeholder.jpg}

    Registering Custom Tags

    Use the goblocks_dynamic_tags filter to add your own tags. Each tag is a callable that receives the tag options array and the current WP_Post object, and must return a string.

    <?php
add_filter( 'goblocks_dynamic_tags', function( array $tags ): array {
    $tags['reading_time'] = function( array $options, WP_Post $post ): string {
        $words   = str_word_count( strip_tags( $post->post_content ) );
        $minutes = (int) ceil( $words / 200 );
        $suffix  = $options['suffix'] ?? 'min read';
        return $minutes . ' ' . $suffix;
    };
    return $tags;
} );

    Usage in the editor:

    {reading_time}
{reading_time|suffix:min}

    Security: The return value of a custom tag is escaped with esc_html() before output. If your tag returns HTML (e.g. an image), escape it appropriately yourself and use wp_kses_post() output escaping in your block's PHP render callback.

    When Tags Are Resolved

    Tags are resolved server-side in the block's PHP render callback — they never appear in the HTML source as raw {…} strings. In the block editor preview, the tag slug is shown as a placeholder; the resolved value is only visible on the frontend.
